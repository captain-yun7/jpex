## 1. 네트워크 기초 (약 5분 설명)

### 핵심 포인트
- **왜 중요한가**: 클라우드든 온프레미스든 네트워크는 모든 인프라의 기반
- **학습 흐름**: OSI 7계층 → TCP/IP → 스위칭 → 라우팅 → AWS VPC

### 주요 학습 내용
```
기초 개념                          AWS 연계
─────────────────────────────────────────────────────
TCP/IP, Protocols          →      VPC 기본 이해
L2/L3/L4/L7 스위치         →      Security Group, NACL
VLAN, Inter-VLAN Routing   →      Subnet 분리
Gateway 이중화(HSRP/VRRP)  →      NAT Gateway
Static/Dynamic Routing     →      Route Table
ACL, NAT                   →      IGW, NAT Gateway
```

### 설명 포인트
> "온프레미스에서 배우는 네트워크 개념이 그대로 AWS VPC 설계에 적용됩니다. VLAN이 Subnet으로, ACL이 Security Group/NACL로, 라우팅 테이블은 동일한 개념입니다."

## 2. 서버 운영체제 - Windows & Linux (약 5분 설명)

### 핵심 포인트
- **Windows**: 기업 환경의 AD, 파일서버, IIS 등 이해
- **Linux**: 클라우드/컨테이너 환경의 필수 OS

### 주요 학습 내용
```
Windows Server                     Linux
─────────────────────────────────────────────────────
GUI/Core 설치               리눅스 설치 및 환경구성
Active Directory            사용자/그룹 관리
IIS 웹서버                  APM(Apache, PHP, MySQL)
DFS 파일서버                NFS, FTP
VPN (IPSec, SSL)            SSH 원격접속
CA 인증기관                 쉘 스크립트
```

### AWS 연계
- **EC2 인스턴스**: Windows/Linux AMI 선택 및 구성
- **실습**: EC2 생성 → 웹서버 구축 → 사용자 정의 AMI 생성

### 설명 포인트
> "Linux 명령어와 쉘 스크립트는 이후 Docker, Kubernetes, 자동화 스크립트의 기반이 됩니다. 반드시 익숙해져야 합니다."

## 3. 데이터베이스 (약 3분 설명)

### 핵심 포인트
- **RDBMS vs NoSQL**: 언제 무엇을 선택할 것인가
- **AWS 매니지드 서비스**: 운영 부담 감소

### 주요 학습 내용
```
온프레미스                         AWS 서비스
─────────────────────────────────────────────────────
RDBMS 설치/설정            →      Amazon RDS
쿼리 작성/최적화           →      RDS 성능 인사이트
트랜잭션, 인덱스           →      RDS 파라미터 그룹
NoSQL 개념                 →      DynamoDB
캐싱                       →      ElastiCache
```

### 설명 포인트
> "클라우드에서는 DB 설치/패치/백업을 직접 하지 않습니다. RDS를 사용하면 이런 운영 업무가 자동화됩니다."

## 4. 클라우드 자동화를 위한 Python (약 7분 설명)

### 핵심 포인트
- **왜 Python인가**: AWS SDK(Boto3), 자동화 스크립트, AI/ML 연계
- **LangChain**: 생성형 AI 활용의 핵심 프레임워크

### 학습 로드맵
```
1단계: Python 기초
   └─ 변수, 자료형, 조건문, 반복문, 함수

2단계: Python 심화
   └─ 모듈, 패키지, PIP, 가상환경

3단계: AI 활용 (LangChain)
   └─ LLM 개요 → LangChain → Amazon Bedrock
   └─ PDF/Word 챗봇, YouTube 요약

4단계: 웹 & 서버리스
   └─ Django 프레임워크
   └─ AWS Lambda (Python으로 EC2 제어)
```

### 설명 포인트
> "Python은 단순 스크립트를 넘어 AI 챗봇까지 만들 수 있습니다. LangChain + Amazon Bedrock 조합으로 실무에서 바로 쓸 수 있는 AI 애플리케이션을 개발합니다."

## 5. KVM 및 Docker 가상화 (약 5분 설명)

### 핵심 포인트
- **KVM**: 전통적인 하이퍼바이저 가상화
- **Docker**: 컨테이너 기반 경량 가상화 (현재 업계 표준)

### Docker 핵심 개념
```
개념                              실무 활용
─────────────────────────────────────────────────────
Container 기본 개념        →      애플리케이션 패키징
Docker Image               →      재사용 가능한 환경
Docker Storage             →      데이터 영속성
Docker Network             →      컨테이너 간 통신
Docker Compose             →      멀티 컨테이너 관리
```

### AWS 연계
- **ECR (Elastic Container Registry)**: Docker 이미지 저장소
- **ECS/EKS**: 컨테이너 오케스트레이션

### 설명 포인트
> "VM은 OS 전체를 가상화하지만, 컨테이너는 애플리케이션만 격리합니다. 훨씬 가볍고 빠릅니다. Docker를 모르면 현대 클라우드 아키텍처를 이해할 수 없습니다."

## 6. 쿠버네티스 아키텍처 (약 7분 설명)

### 핵심 포인트
- **왜 Kubernetes인가**: 컨테이너가 수십, 수백 개일 때 어떻게 관리할 것인가
- **핵심 컴포넌트**: Pod, Service, Deployment, Ingress

### 학습 로드맵
```
1단계: 아키텍처 이해
   └─ Master Node / Worker Node
   └─ etcd, API Server, Scheduler, Controller

2단계: 워크로드 배포
   └─ Pod → ReplicaSet → Deployment
   └─ 롤링 업데이트, 롤백

3단계: 서비스 노출
   └─ ClusterIP → NodePort → LoadBalancer
   └─ Ingress Controller

4단계: 스토리지 & 보안
   └─ PV, PVC (영구 볼륨)
   └─ RBAC (역할 기반 접근 제어)

5단계: 오토스케일링
   └─ HPA (Horizontal Pod Autoscaler)
```

### AWS 연계
- **Amazon EKS**: 관리형 Kubernetes 서비스
- **실습**: EKS로 웹 애플리케이션 구축

### 설명 포인트
> "Kubernetes는 러닝커브가 있지만, 한번 익히면 어떤 클라우드에서든 동일하게 사용할 수 있습니다. AWS EKS, GCP GKE, Azure AKS 모두 동일한 K8s입니다."

## 9~12. AWS 클라우드 (약 15분 설명 - 가장 중요)

### 학습 단계

#### 9단계: AWS 기초 (TESS)
```
핵심 서비스 6가지:
├── IAM: 보안 및 접근 제어
├── EC2: 가상 서버
├── VPC: 네트워크
├── S3: 객체 스토리지
├── DynamoDB: NoSQL 데이터베이스
└── ELB + Auto Scaling: 고가용성
```

#### 10단계: AWS 아키텍처 설계 (ARC)
```
Well-Architected Framework 기반:
├── 보안: IAM, 다중 계정 관리
├── 네트워킹: VPC 심화, 엔드포인트
├── 컴퓨팅: EC2 인스턴스 유형, 요금
├── 스토리지: S3, EFS, FSx
├── 데이터베이스: RDS, DynamoDB, 캐싱
├── 자동화: CloudFormation, Beanstalk
├── 컨테이너: ECR, ECS, EKS
├── 서버리스: Lambda, API Gateway, SQS, SNS
└── 엣지 서비스: Route53, CloudFront
```

#### 11단계: AWS 고급 아키텍처 (AARC)
```
엔터프라이즈 수준:
├── 멀티 계정: Control Tower
├── 하이브리드: VPN, Direct Connect
├── 보안: WAF, Shield, KMS
├── 데이터 레이크: Lake Formation
├── 마이그레이션: DataSync, Storage Gateway
└── 비용 최적화
```

#### 12단계: DevOps CI/CD
```
AWS Code 시리즈:
├── CodeCommit: 소스 관리
├── CodeBuild: 빌드
├── CodeDeploy: 배포
├── CodePipeline: 파이프라인 통합
└── Blue/Green 배포 전략
```

### 설명 포인트
> "AWS 자격증으로 치면 SAA(Solutions Architect Associate)부터 SAP(Professional) 수준까지 다룹니다. 실습 위주로 진행되며, 실제 아키텍처를 직접 구축합니다."

## 13~15. Google Cloud (약 7분 설명)

### 핵심 포인트
- **멀티 클라우드 역량**: AWS만으로는 부족
- **GKE**: Google이 Kubernetes를 만들었음 (가장 성숙한 관리형 K8s)
- **Terraform**: 멀티 클라우드 IaC 표준

### 학습 내용

#### 13단계: GCP 핵심 서비스
```
AWS                               GCP
─────────────────────────────────────────────────────
EC2                        →      Compute Engine
VPC                        →      VPC Network
S3                         →      Cloud Storage
RDS                        →      Cloud SQL
EKS                        →      GKE
Lambda                     →      Cloud Run / Cloud Functions
```

#### 14단계: GKE (Google Kubernetes Engine)
```
- GKE 클러스터 배포 및 업그레이드
- Deployments, Jobs, 오토스케일링
- 서비스 및 인그레스
- 영구 스토리지 (PV, PVC)
- IAM 및 RBAC 보안
- Monitoring & Logging
- CI/CD with Cloud Build
```

#### 15단계: Terraform
```
- IaC(Infrastructure as Code) 개념
- Terraform 상태 관리
- 모듈화 및 재사용
- 멀티 클라우드 배포 자동화
```

### 설명 포인트
> "Terraform을 익히면 AWS든 GCP든 Azure든 동일한 코드로 인프라를 관리할 수 있습니다. 이것이 진정한 멀티 클라우드 역량입니다."
