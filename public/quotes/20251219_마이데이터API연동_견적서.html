마이데이터 API 연동 개요

참조 규격: 금융분야 마이데이터 표준 API 규격 (금융보안원)

관련 법규: 신용정보법, 개인정보보호법, 특정금융정보법

테스트베드: 금융분야 MYDATA TESTBED

가상자산사업자(VASP)가 마이데이터 사업자에게 개인신용정보를 제공하기 위한 정보제공자 API 시스템 개발

마이데이터 2.0 주요 변경사항 (2025년 적용)
  • 전송요구 동의 절차 2단계 → 1단계 간소화
  • 정기적 전송이 기본값으로 설정
  • API v2 규격 적용 (2025-09-30까지 v1/v2 병행)
  • 개인(신용)정보 제3자 제공 동의 내역 철회 요청 API 추가

기술 규격 요구사항

통신 보안 규격
  • TLS 1.3 이상: 전송 구간 암호화 필수
  • mTLS (Mutual TLS): 양방향 인증서 기반 상호인증
  • REST API: HTTP 메소드 GET, POST만 허용
  • JWS (JSON Web Signature): 접근토큰 무결성 보장
OAuth 2.0 인증 규격
  • 접근토큰 (Access Token): 유효기간 최장 90일, JWS 규격
  • 리프레시 토큰 (Refresh Token): 유효기간 최장 1년
  • Scope: 업권별 자산목록 scope 필수 포함
  • 토큰 발급: Authorization Code Grant 방식
기술 스택

Java/Spring Boot OAuth 2.0 JWS/JWT mTLS PostgreSQL

개발 범위

A. 정보제공 API 개발
  • 접근토큰 발급/갱신/폐기 API
  • 자산목록 조회 API (가상자산 보유 현황)
  • 거래내역 조회 API (입출금, 매매 내역)
  • 기본정보 조회 API (계정 정보)
  • 전송요구 내역 조회/철회 API
B. 인증/보안 시스템
  • OAuth 2.0 인증 서버 구축
  • mTLS 상호인증 구현
  • JWS 토큰 발급/검증 모듈
  • API 키/시크릿 관리 시스템
  • 접근 로그 기록 및 감사 추적
C. 데이터 연동
  • 기존 가상자산 시스템 데이터 연동
  • 마이데이터 표준 포맷 변환 모듈
  • 실시간/배치 데이터 동기화
  • 개인정보 암호화 처리 (AES-256)
D. 관리 시스템
  • API 모니터링 대시보드
  • API 호출 통계 및 과금 관리
  • 장애 알림 시스템
  • 마이데이터사업자 등록/관리
E. 테스트/인증
  • 마이데이터 테스트베드 연동 테스트
  • 금융보안원 적합성 검증 대응
  • 성능/부하 테스트
  • 보안 취약점 점검
